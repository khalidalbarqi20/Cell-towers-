# مستكشف الأبراج (Cell Tower Explorer)

تطبيق أندرويد خاص للاستخدام الشخصي (غير مخصص للنشر في المتجر)، يعرض الأبراج
القريبة على خريطة، يقرأ بيانات الشبكة من شريحة هاتفك، ويبني قاعدة بيانات
محلية تتحسن تدريجياً مع استخدامك للتطبيق.

## طريقة البناء من الجوال فقط (بدون كمبيوتر) — موصى بها لك

المشروع مُعَدّ ليُبنى تلقائياً عبر **GitHub Actions** (خدمة GitHub المجانية
لتشغيل أوامر البناء على سيرفراتها). كل ما تحتاجه: حساب GitHub، وتطبيق
Termux على جوالك لرفع الملفات (أو رفعها يدوياً من متصفح الجوال). ملف
`.github/workflows/build.yml` جاهز فعلاً داخل المشروع ويقوم بكل خطوات
البناء تلقائياً عند رفع الكود.

الخطوات بالتفصيل موجودة في الرسالة التي أرسلها لك المساعد في المحادثة،
وتلخيصها:
1. أنشئ مستودع (Repository) جديد فاضي في GitHub من متصفح الجوال.
2. ثبّت Termux (من F-Droid، لا تستخدم نسخة Google Play القديمة)، وارفع
   محتوى هذا المشروع له عبر `git push`.
3. أضف مفتاح OpenCellID كـ **GitHub Secret** باسم `OPENCELLID_API_KEY`
   (Settings → Secrets and variables → Actions → New repository secret)
   — لا تكتبه داخل local.properties عند هذي الطريقة.
4. من تبويب Actions بالمستودع، انتظر اكتمال البناء، ثم نزّل ملف APK من
   "Artifacts" وثبّته على جوالك.

## طريقة البناء عبر Android Studio (إذا توفر كمبيوتر مستقبلاً)

1. افتح Android Studio (إصدار حديث، Hedgehog أو أحدث يفضّل).
2. اختر "Open" ووجّهه لمجلد `CellTowerExplorer` بالكامل.
3. اترك Android Studio يقوم بمزامنة Gradle تلقائياً (Sync Now). إذا اقترح
   ترقية إصدار AGP/Gradle فاقبل الاقتراح، عادة يحل أي تعارض إصدارات بسيط.

## مفتاح OpenCellID (مطلوب)

التطبيق يحتاج مفتاح API مجاني من OpenCellID لجلب الأبراج من قاعدة بياناتهم:

1. سجّل حساباً مجانياً في https://opencellid.org
2. من لوحة التحكم، انسخ "API Key" الخاص بك.
3. افتح ملف `local.properties` في جذر المشروع، وضع المفتاح بهذا الشكل:
   ```
   OPENCELLID_API_KEY=مفتاحك_الحقيقي_هنا
   ```
4. أعد مزامنة Gradle بعد التعديل.

هذا المفتاح لا يُكتب داخل الكود مباشرة ولا يُرفع لأي مستودع Git (الملف
`local.properties` مستثنى تلقائياً من Git في مشاريع أندرويد).

## الصلاحيات التي سيطلبها التطبيق

الموقع الدقيق (GPS)، وحالة الهاتف (لقراءة بيانات الخلية)، وإشعارات (لإظهار
إشعار خدمة التتبع في الخلفية على أندرويد 13+). كل الصلاحيات تُستخدم محلياً
فقط ولا تُرسل لأي خادم خارجي سوى استعلامات OpenCellID العامة.

## ماذا يعمل الآن

- خريطة OpenStreetMap مع دائرة نطاق قابلة للتغيير (1/5/10 كم).
- البحث المحلي أولاً، ثم OpenCellID عند الحاجة، مع تخزين النتائج محلياً.
- بطاقة علوية تعرض البرج المتصل به هاتفك حالياً (Cell ID، PCI، الإشارة...)
  مع تحديث تلقائي كل 5 ثوانٍ.
- الضغط على أي علامة على الخريطة يعرض تفاصيلها (المشغّل، المسافة، الاتجاه
  بالعربية، درجة الثقة، آخر تحديث).
- زر "ابدأ التتبع" يشغّل خدمة أمامية (Foreground Service) تسجّل موقعك
  وبيانات الخلية كل 10 ثوانٍ في قاعدة بيانات محلية (Room) حتى لو كان
  التطبيق بالخلفية، مع إشعار دائم كما يتطلبه أندرويد.
- شاشة إحصائيات: إجمالي الأبراج، إجمالي القراءات، عدد أبراج 4G/5G، وتوزيع
  الأبراج حسب المشغّل.
- تصفية النتائج حسب المشغّل (STC/Mobily/Zain) ونوع الشبكة (4G/5G).
- معالجة آمنة للحقول غير المتوفرة (PCI، NRARFCN، الترددات...) فبعض الأجهزة
  أو إصدارات أندرويد لا تكشف عنها — لن يتعطل التطبيق في هذه الحالة.

## حدود معروفة يجب أخذها بالحسبان

- **حد OpenCellID**: كل طلب يعيد حتى 50 برجاً فقط (limit). لمنطقة مزدحمة
  بأكثر من 50 برج ضمن النطاق المختار، النتائج الحالية لا تُكمل الصفحات
  التالية (offset) تلقائياً. هذه إضافة بسيطة إذا احتجتها لاحقاً.
- **الحصة اليومية**: خدمة OpenCellID المجانية محدودة بعدد طلبات يومي،
  وقد تحتاج لاحقاً لتسجيل تطبيقك إن واجهت رفض طلبات متكرر.
- **PCI من OpenCellID**: قاعدة بيانات OpenCellID لا تعيد PCI ضمن نتائج
  البحث، فهو يظهر فقط للبرج المتصل به فعلياً عبر قراءة الشريحة المباشرة
  (البطاقة العلوية)، وليس لكل الأبراج المعروضة على الخريطة. هذا متوقع
  وليس خللاً.
- **بدون توقيع/نشر**: هذا بناء تجريبي شخصي (Debug build) فقط، لا حاجة
  لأي خطوات توقيع أو نشر على المتجر.
- لم يتم تجربة بناء المشروع فعلياً (لا تتوفر بيئة أندرويد ستوديو/Gradle في
  بيئة التطوير التي كتبت بها الكود)، فمن المتوقع وجود تعديلات صغيرة قد
  تحتاجها عند أول مزامنة/بناء فعلي (مثل توافق إصدار AGP/Gradle مع جهازك).

## المرحلة المستقبلية (غير مطلوبة الآن، حسب طلبك)

خوارزمية تقدير موقع البرج بناءً على Signal Strength + GPS + Trilateration
لتحسين دقة قاعدة البيانات المحلية تدريجياً في المناطق التي تتكرر زيارتها،
لتتجاوز دقة OpenCellID مع الوقت. الجداول الحالية (`towers`/`observations`)
ودرجة الثقة (`ConfidenceCalculator`) مصمَّمة لتستوعب هذه الميزة مستقبلاً
دون تغييرات جوهرية في قاعدة البيانات.
